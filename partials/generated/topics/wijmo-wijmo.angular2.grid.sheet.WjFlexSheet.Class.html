Angular 2 组件用于 FlexSheet 控件 。

使用 wj-flex-sheet 组件向你的 Angular2 应用添加 FlexSheet 控件。关于跟多 Angular 2标记语言，请参见Angular 2 Markup。

WjFlexSheet 组件继承自 FlexSheet 控件，继承了全部的属性，事件和方法 。

The wj-flex-sheet component may contain WjSheet child components.

itemFormatter Function

获取或设置一个格式化函数，用做自定义该表单上的单元格。

该格式化函数可以添加任意的内容至任意的单元格。这将提供了对表单单元格外观和行为完全灵活的定制能力。

如果指定，该函数应当接受四个参数：包含该单元格的 GridPanel，该单元格的行索引和列索引，以及表示该单元格的 HTML 元素。 该函数通常将修改单元格元素的 innerHTML 属性。

例如： 

flex.itemFormatter = function(panel, r, c, cell) {
  if (panel.cellType == CellType.Cell) {
    // 在单元格上绘制 sparkline
    var col = panel.columns[c];
    if (col.name == 'sparklines') {
      cell.innerHTML = getSparklike(panel, r, c);
    }
  }
}

请注意 FlexGrid 将回收并重复使用单元格实例，因此如果您的 itemFormatter 更改了单元格的样式属性，您必须确保在其他不需要这些样式属性的单元格中已经重置了 这些设置。例如：

flex.itemFormatter = function(panel, r, c, cell) {
  // 重置掉我们将自定义的全部属性
  var s = cell.style;
  s.color = '';
  s.backgroundColor = '';
  // 自定义单元格的颜色和背景色属性
  ...
}

如果您需要多个客户端自定义表单呈现的场景 （比如说当创建 directive 或者可重用的库），请考虑使用 formatItem 事件。 该事件允许多个不同的客户端关联各自不同的处理逻辑。

在 FlexGrid 中进行编辑和在 Excel 中进行编辑类似： 按下 F2 或者双击一个单元格将使得表单进入完全编辑模式。 在该模式下，单元格编辑器将始终保持活动状态，直到用户按下了 Enter，Tab，或者 Escape 键。 再或者当用户通过鼠标移动选择至其他单元格。在完全编辑模式下，按下方向键将不会使得表单 离开编辑模式。

完全编辑模式通常用做对当前值进行修改。 快速编辑模式通常用做快速地输入新值。
