为 FlexGrid 控件实现一个 Excel 样式的筛选器。

为了在 FlexGrid 控件上启用筛选，需要创建一个 FlexGridFilter 的实例， 并将 grid 作为一个参数传递给构造器。例如：

// 创建 FlexGrid
var flex = new wijmo.grid.FlexGrid('#gridElement');
// 在 FlexGrid 上启用筛选
var filter = new wijmo.grid.filter.FlexGridFilter(flex);

一旦这一过程完成，则表单的列头上将添加一个筛选器图表。 单击该图表将显示一个编辑器，用户可以在此编辑该列的筛选 条件。

该 FlexGridFilter 类依赖 wijmo.grid 以及 wijmo.input 模块。

属性

名称 类型 描述
defaultFilterType FilterType

获取或设置所使用的默认筛选器类型。

该值可以在指定列的筛选器上被覆盖。例如，以下代码创建一个筛选器，在全部的列上 按照指定条件筛选，除了“ByValue”列以外：

var f = new wijmo.grid.filter.FlexGridFilter(flex);
f.defaultFilterType = wijmo.grid.filter.FilterType.Condition;
var col = flex.columns.getColumn('ByValue'),
    cf = f.getColumnFilter(col);
cf.filterType = wijmo.grid.filter.FilterType.Value;

filterColumns string[]

获取或设置一个包含具有筛选器的列的名称或绑定的数组。

设置该属性为 null 或者一个空的数组将添加筛选器至全部的列。
filterDefinition string 获取或设置以 JSON 字符串表示的当前的筛选器定义。
grid FlexGrid 获取拥有该筛选器的 FlexGrid 的一个引用。
showFilterIcons boolean

获取或设置一个布尔值，该值表示是否 FlexGridFilter 向表单的列头添加编辑按钮。

如果您设置该属性的值为 false，那么您需要自行向用户提供编辑/清除/应用筛选器的方式。
showSortButtons boolean

获取或设置一个布尔值，该值表示是否筛选编辑器应当包含排序按钮。

默认情况下，编辑器和 Excel 一样提供排序按钮。但是由于用户也可以通过单击列头进行排序， 在某些场景下排序按钮不是那么的有迫切需求。

事件

名称 参数 描述
filterApplied EventArgs 在筛选器被应用之后触发。
filterChanged CellRangeEventArgs

在一个列筛选器被用户编辑完成之后触发。

使用该事件参数获取拥有该筛选器的列，并确定是否该变更被应用或取消。
filterChanging CellRangeEventArgs

当一个列筛选器即将被用户编辑时触发。

如果您希望覆盖筛选器默认的设置，您可以通过该事件自定义该列筛选器。

比如说，以下代码当筛选器条件使用的操作为符 null 时，设置该操作符为 ‘contains’：

filter.filterChanging.addHandler(function (s, e) {
  var cf = filter.getColumnFilter(e.col);
  if (!cf.valueFilter.isActive && cf.conditionFilter.condition1.operator == null) {
    cf.filterType = wijmo.grid.filter.FilterType.Condition;
    cf.conditionFilter.condition1.operator = wijmo.grid.filter.Operator.CT;
  }
});
