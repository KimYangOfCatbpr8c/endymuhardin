wijmo.js

描述一个事件。

Wijmo 事件相似于.NET 事件。任何类可能将它们声明为字段来定义事件。任何类可以使用事件的addHandler 方法来订阅事件，也可以使用removeHandler 方法取消订阅。

Wijmo 事件处理器有两个参数: sender 和 args. 第一个参数是引发事件的对象，而第二个参数是包含事件参数的对象。

根据.NET模型为每一个事件定义事件的类。on[EVENTNAME] 方法引发事件。该模式允许衍生的来重载on[EVENTNAME] 方法，并且在引发基础类之前或者之后处理事件。 衍生的类可以甚至通过不调用基础类来阻止事件的实现。

例如，下面类脚本化的代码重载控件的 onValueChanged 事件，在执行一些处理之前且在valueChanged事件触发之后：

// 重载基础类
onValueChanged(e: EventArgs) {
  // 在事件触发前执行一些代码
  console.log('about to fire valueChanged');
  // 可选择地，调用基础类来触发事件
  super.onValueChanged(e);
  // 在事件触发之后执行一些代码
  console.log('valueChanged event just fired');
}
