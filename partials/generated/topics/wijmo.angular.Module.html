模块 wijmo.angular

文件
wijmo.angular.js

为 Wijmo 控件包含 AngularJS 指令。

这些指令允许在 HTML 页面中使用简单的标记添加 Wijmo 控件到 AngularJS 应用中。

可以像常规的 HTML 标记那样使用这些指令，标签的名字与控件名称相对应，添加了前缀 "wj-," 并且属性名称和事件名称都是相互对应的。

所有的控件，属性，和事件名称都使用约定的横线分隔的小写替代驼峰命名。

AngularJS 有3种指令参数，依赖于使用的绑定类型，下表对其进行了描述:

@
值绑定，即单向绑定，属性值被解释为字面量。
=
引用绑定，即双向绑定。属性值被解释为表达式。
&
函数绑定。 属性值被解释为带有参数的回调函数。

关于绑定类型的更多细节，请参见 Dan Wahlin's blog on directives. 文档没有描述指令事件，因为其通常与控件事件致，并且绑定机制也相同（函数绑定）。

举例说明，这是用于创建 ComboBox 控件的标签:

<wj-combo-box 
  text="ctx.theCountry"
  items-source="ctx.countries"
  is-editable="true"
  selected-index-changed="ctx.selChanged(s, e)">
</wj-combo-box>

注意 ComboBox 的 text 属性绑定到控制器的 "ctx.theCountry" 属性上，该绑定使用双向绑定；控件和值域的更新会互相作用。初始化 text 属性是使用的是单引号中的字符常量(例如, text="'constant'").

注意 selected-index-changed 事件绑定到控件 "selChanged"，并且绑定包含两个事件参数(没有参数，该函数不会被调用). 任何时候只要控件触发了事件，指令就会调用控制器方法。

WjAutoComplete

AngularJS 指令应用于 AutoComplete 控件.

使用 wj-auto-complete 指令添加 AutoComplete 控件到 AngularJS 应用中。 注意指令和参数名称必须使用小写字母+短线语法替代驼峰命名。 示例：

<p>Here is an AutoComplete control:</p>
<wj-auto-complete
  text="theCountry" 
  items-source="countries"
  is-editable="false" 
  placeholder="country">
</wj-auto-complete>

下面的例子创建了一个 AutoComplete 控件，并绑定到控制器公开的 'countries' 数组上。 AutoComplete 根据用户类型检索country，在列表项中自动匹配当前输入。

在线示例

css-match

@ 键盘输入后延迟执行检索的毫秒数。

= 动态提供用户类型数据源的函数。
max-items
@ 下拉列表最多显示多少项
min-length
@ 触发自动检索执行所需输入的最小字符数量.

WjFlexGridCellTemplate

AngularJS 指令应用于 FlexGrid 单元格模板。

wj-flex-grid-cell-template 指令定义 FlexGrid 确定的单元格类型的模板, 并且必须使用 CellTemplateType 指定 cell-type 属性。 依赖于模板的单元格类型，wj-flex-grid-cell-template 指令必须包含一个子指令，WjFlexGrid 或者 WjFlexGridColumn 指令。

指定列的模板必须包含于 wj-flex-grid-column 指令中，并且所有不符合指定列的单元格（如行头或左上单元格）必须包含于 wj-flex-grid 指令中。

为了添加 HTML 代码段， wj-flex-grid-cell-template 指令可以包含 ng-style or ng-class 属性用以提供单元的条件格式化。

ng-style/ng-class 属性和 HTML 代码段都可以 $col, $row and $item使用模板变量引用 Column, Row 和 Row.dataItem 对象，以及其上的单元格。

对于这些单元格类型如 Group 和 CellEdit, 附加的 $value 变量包含一个未被格式化的单元格值。示例是一个 FlexGrid ，为 Country 列常规和列头单元格使用了不同的模板:

<wj-flex-grid items-source="data">
  <wj-flex-grid-cell-template cell-type="RowHeader">
    {​{$row.index}}
  </wj-flex-grid-cell-template>
  <wj-flex-grid-cell-template cell-type="RowHeaderEdit">
    ...
  </wj-flex-grid-cell-template>
 
  <wj-flex-grid-column header="Country" binding="country">
    <wj-flex-grid-cell-template cell-type="ColumnHeader">
      <img ng-src="resources/globe.png" />
        {​{$col.header}}
      </wj-flex-grid-cell-template>
      <wj-flex-grid-cell-template cell-type="Cell">
        <img ng-src="resources/{​{$item.country}}.png" />
        {​{$item.country}}
      </wj-flex-grid-cell-template>
    </wj-flex-grid-column>
  <wj-flex-grid-column header="Sales" binding="sales"></wj-flex-grid-column>
</wj-flex-grid>

关于单元格模板更多细节请参见 CellTemplateType 枚举.

WjRange

AngularJS 指令应用于 Range 对象。

<wj-radial-gauge

    <wj-range

WjValidationError

使用 AngularJS 指令创建基于表达式的自定义验证器。

wj-validation-error 指令同时支持AngularJS和原生HTML5验证机制。它接受一个AngularJS表达式，该表达式在输入无效时返回错误消息的字符串，在输入有效时返回empty字符串。

为了使用AngularJS验证，必须同时应用 ng-model 指令。 wj-validation-error 指令报告一个错误消息，调用 NgModelController.$setValidity 方法，并使用 wjValidationError 错误标识key, AngularJS 原生和自定义验证指令使用相同的机制。

对于HTML5验证, wj-validation-error 指令会用HTML5的验证API提供的 setCustomValidity 方法为元素设置一个错误状态。 示例：

<p>HTML5 validation:</p>
<form>
    <input type="password"
        placeholder="Password" 
        name="pwd" ng-model="thePwd" 
        required minlength="2" />
    <input type="password"
        placeholder="Check Password"
        name="chk" ng-model="chkPwd"
        wj-validation-error="chkPwd != thePwd ? 'Passwords don\'t match' : ''" />
</form>



<p>AngularJS validation:</p>
<form name="ngForm" novalidate>
    <input type="password"
        placeholder="Password" 
        name="pwd" ng-model="thePwd" 
        required minlength="2" />
    <input type="password"
        placeholder="Check Password"
        name="chk" ng-model="chkPwd"
        wj-validation-error="chkPwd != thePwd" />
    <div 
        ng-show="ngForm.chk.$error.wjValidationError && !ngForm.chk.$pristine">
        Sorry, the passwords don't match.
    </div>
</form>
