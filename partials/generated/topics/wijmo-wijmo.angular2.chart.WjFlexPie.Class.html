FlexPie控件对应的 Angular 2 组件。

使用 wj-flex-pie 组件添加 FlexPie 控件至您的 Angular 2 应用程序中。关于 Angular 2 标记语法的更多细节，请参见 Angular 2 标记。

WjFlexPie 组件继承自 FlexPie 控件，因此继承了其全部的属性，事件和方法。

wj-flex-pie 组件可以包含 WjFlexChartLegend 以及 WjFlexPieDataLabel 子组件。

plotMargin any

您可以设置该属性为一个数值类型的值或者一个通过 CSS-样式指定的边距。例如：

// 将绘图区的边距四边均设置为 20 个像素

// 分别设置绘图区域边距的顶部，右侧，底部和左侧。

// 分别设置绘图区域边距的顶部/底部为 10 像素，左侧/右侧为 20 像素。
chart.plotMargin = '10 20';

方法

名称 参数 描述
addEventListener​(target, type, fn, capture?)

target (EventTarget): 时间的目标元素。
type (string): 指定该事件的字符串。
fn (any): 当事件发生时执行的函数。
capture? (boolean): 监听是否正在捕获。
(默认值: false)

为Control 上的元素增加一个事件监听。

该控件持有依附于监听和处理器的列表，当控件被处理掉时使它更容易地移除这些监听和处理器（参见dispose 和removeEventListener 方法）。

移除事件监听失败时，可能引起内存泄露。


(继承自 Control).
applyTemplate​(classNames, template, parts, namePart?): HTMLElement

classNames (string): 添加到控件主元素上的类的名称。
template (string): 定义控件模板的HTML字符串。
parts (Object): 部分变量及其名称的字典。
namePart? (string): 部分名称将以主元素命名。 确定控件在表单中使用时如何提交数据。

将这个模板应用到控件的一个新的实例上，并且返回根元素。

这个方法应该被模板化控件的构造器调用。它的职责是绑定模板部分到相应的控件成员上。

例如，下面的代码应用一个模板到InputNumber 控件的实例上。这个模板必须包含有'wj-part'属性的元素，该属性值可以被设置为'input', 'btn-inc'，和'btn-dec'。 该控件的成员'_tbx', '_btnUp', 和 '_btnDn'将被分配一些引用到这些元素上。

this.applyTemplate('wj-control wj-inputnumber', template, {
  _tbx: 'input',
  _btnUp: 'btn-inc',
  _btnDn: 'btn-dec'
}, 'input');

initialize​(options)

options (any): Object that contains the initialization data.

通过从给定的对象中拷贝属性来初始化控件。

该方法允许你使用普通的数据对象代替在代码中为每一个属性设置值来初始化控件。

例如: 

grid.initialize({
  itemsSource: myList,
  autoGenerateColumns: false,
  columns: [
    { binding: 'id', header: 'Code', width: 130 },
    { binding: 'name', header: 'Name', width: 60 } 
  ]
});
// 相当于
grid.itemsSource = myList;
grid.autoGenerateColumns = false;
// 等等.

初始化数据被应用时，它作为一种类型检查。如果初始化的对象包含不知名的属性名称或者无效的数据类型时，该方法将被抛出。

S invalidateAll​(e?)

e? (HTMLElement): Container element. If set to null, all Wijmo controls on the page will be invalidated.

使所有包含在HTML元素中的Wijmo控件无效。

当你的应用包含有动态的面板，且该面板能改变控件的可见性或者面积时，可以使用该方法。例如，splitters，accordions， 和tab控件通常改变控件内容元素的可见性。在这种情况下，通知包含在该元素中的控件失败时，将会引起它们适当地停止工作。

如果此情况发生，你必须处理在动态容器中合适的事件，并且调用Control 使所有的方法无效，这样包含的Wijmo控件将适当地更新它们的布局信息。
