模块 wijmo/wijmo.angular2.input

文件
wijmo.angular2.js

包含 wijmo.input 模块的 Angular 2 组件。

wijmo.angular2.input 是一个外部的 TypeScript 模块，可以通过其环境模块名称导入到您的代码中。例如：

import * as wjInput from 'wijmo/wijmo.angular2.input';
 
@Component({
    directives: [wjInput.WjInputNumber],
    template: '<wj-input-number [(value)]="amount"></wj-input-number>',
    selector: 'my-cmp',
})
export class MyCmp {
    amount = 0;
}

WjContextMenu

上下文菜单的Angular 2 Directive 指令。

使用 wjContextMenu 指令向页面上的元素添加上下文菜单。 wjContextMenu 指令基于 wj-menu 组件； 当用户在一个元素上执行一个上下文菜单请求时（通常为单击鼠标右键），它将显示一个弹出式菜单。

wjContextMenu 指令以一个参数的形式指定，应用到该上下文菜单所要应用的元素上。 该参数的值是一个到 wj-menu 组件的引用。例如：

<!-- 带有上下文菜单的段落 -->
<p [wjContextMenu]="menu" >
 This paragraph has a context menu.</p>



<!-- 定义该上下文菜单（隐藏并且具有 id） -->
<wj-menu #menu style="display:none">

 <wj-menu-item [cmd]="cmdSave" [cmdParam]="2">Save </wj-menu-item>
 <wj-menu-item [cmd]="cmdSave" [cmdParam]="3">Save As...</wj-menu-item>

</wj-menu >

WjMenuItem

菜单项的Angular 2 Directive 指令。

wj-menu-item 组件必须包含在一个 WjMenu Directive 指令中。 它支持以下属性：

[cmd]
当项目被单击时，在 controller 上 执行的函数。
[cmdParam]
当项目被单击时，传递给 cmd 函数 的参数。
[(value)]
当项目单击时，选中的值 （可选地使用此参数和 cmd参数）。

该项目显示的内容可以包含任意的带有 Angular 2 绑定的HTML 片段，组件以及 Directive 指令。 您也可以使用 ngFor 以及 ngIf 指令生成菜单控件中的项。 本地的 item， itemIndex 以及 control 模板变量可以在 Angular 2 绑定中使用，分别表示数据项，当前项的索引以及 owner 控件。
WjMenuSeparator

对应菜单分隔线的 Angular 2 组件。

wj-menu-item-separator 组件必须包含在一个 WjMenu Directive 指令中。 它将添加一个不可选的分隔线至菜单，并且不具有任何 attribute 属性。
