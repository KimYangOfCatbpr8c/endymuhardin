Calendar控件对应的 Angular 2 组件。

使用 wj-calendar 组件添加 Calendar 控件至您的 Angular 2 应用程序中。关于 Angular 2 标记语法的更多细节，请参见 Angular 2 标记。

WjCalendar 组件继承自 Calendar 控件，因此继承了其全部的属性，事件和方法。

itemFormatter Function

获取或设置一个 formatter 函数，用做自定义日历上的日期。

formatter 函数可以向任意日期添加任意内容。它允许对日历的外观和行为做完全的自定义。

如果指定了该函数，该函数接受两个参数：

  • 即将被格式化的日期
  • 用来表示该日期的 HTML 元素

例如，以下代码将使得周末具有一个黄颜色的背景： 

calendar.itemFormatter = function(date, element) {
  var day = date.getDay();
  element.style.backgroundColor = day == 0 || day == 6 ? 'yellow' : '';
}


(继承自 Calendar).
itemValidator Function

获取或设置一个校验函数，用来决定是否日期可以被合法地选择。

如果指定了该函数，则该 validator 函数应当接受一个参数，表示 待测试的日期，如果该日期非法或者不应当被选中，则应当返回 false。

例如，以下代码将周末显示为禁用状态，并防止用户 选中这些日期： 

calendar.itemValidator = function(date) {
  var weekday = date.getDay();
  return weekday != 0 && weekday != 6;
}

S invalidateAll​(e?)

e? (HTMLElement): Container element. If set to null, all Wijmo controls on the page will be invalidated.

使所有包含在HTML元素中的Wijmo控件无效。

当你的应用包含有动态的面板，且该面板能改变控件的可见性或者面积时，可以使用该方法。例如，splitters，accordions， 和tab控件通常改变控件内容元素的可见性。在这种情况下，通知包含在该元素中的控件失败时，将会引起它们适当地停止工作。

如果此情况发生，你必须处理在动态容器中合适的事件，并且调用Control 使所有的方法无效，这样包含的Wijmo控件将适当地更新它们的布局信息。

formatItem FormatItemEventArgs

当表示一天的一个元素被创建时触发。

该事件可以用来格式化显示的日历项。 从目的上看，该属性和 itemFormatter 属性的功能类似，但是该属性具有 允许多个独立的处理器的额外好处。

例如，以下代码使用 formatItem 事件禁用周末， 因此它们在日历上显示为灰色：

// 禁用周六和周日
calendar.formatItem.addHandler(function (s, e) {
  var day = e.data.getDay();
  if (day == 0 || day == 6) {
    wijmo.addClass(e.item, 'wj-state-disabled');
  }
});
