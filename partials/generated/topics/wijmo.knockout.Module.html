模块 wijmo.knockout

文件
wijmo.knockout.js

Wijmo控件拥有使用KnockoutJS绑定的特性.该绑定特性允许你在HTML页面中使用简单的标记将Wijmo控件引入到 KnockoutJS 应用中。将Wijmo控件添加到页面标记中，添加 <div> 元素，并使用 data-bind 属性定义该绑定特性。绑定控件的名称是对应的控件名称加上wj前缀。例如，Wijmo的wjInputNumber 绑定控件对应控件InputNumber。 绑定的值是一个包含一系列控件读写属性与事件名称的对象字面量，并且他们的值与相对应的控件中定义的属性值和事件相一致。下面的标记创建了一个Wijimo InputNumber 控件，并且将 value 属性绑定到视图模型view model的 theValue 属性上, 将 step 属性设置为1，inputType 属性设置为 'text':

<div data-bind="wjInputNumber: { value: theValue, step: 1, inputType: 'text' }"></div>

定制元素

在标准的Knockout绑定语法之外，Wijimo为Knockout提供了在页面中使用的定制元素标签，这些元素标签对应其控件的绑定名称，其标签属性对应其控件的相应属性，这些元素和其参量名称需要格式化为纯小写，并使用连字符（-）替换原来的驼峰命名。上边的例子中的控件可以使用定制元素语法定义如下:

<wj-input-number value="theValue" step="1" input-type="'text'"></wj-input-number>

注意属性值的定义需要与你的data-bind定义严格对应，并且符合JavaScript表达式语法规则。Wijmo的Knockout预处理器会将这些定制元素转换为常规的data-bind形式。更多细节请参见主题 定制元素预处理器 Custom elements preprocessor 。

绑定控件属性

Wijmo 的 KnockoutJS 绑定支持绑定到控件的任何可读可写属性上。你可以为属性指定任何有效的 KnockoutJS 表达式（如常量、 视图模型的监控属性或复杂的表达式）。大多数属性提供了单向绑定，即视图模型上的监控属性更改可以触发到其对应的绑定控件上相应属性的更改，反之则不会触发。另外一些属性支持双向绑定，这意味着绑定控件属性的更改也可以传播到其对应的视图模型的监控属性上。双向绑定用于这类属性，即通过用户交互或其他操作可被控件自身更改的属性。 例如，InputNumber 控件为 value and text 属性提供了双向绑定, 该属性在用户输入了新的值后会被控件更改， InputNumber 控件的其他属性工作在单向绑定模式下。

绑定控件实践

要跟踪事件处理，需要在定义控件绑定的对象字面量上定义一个属性，属性名为事件名称，以及一个在该事件触发时调用的函数，该函数作为属性的值。 Wijmo绑定在定义事件处理上遵循与KnockoutJS绑定一致的规则。例如 click 和 event，事件处理接收如下的一组特定顺序的参数:
  • data: 当前模型的值，与KnockoutJS绑定原生的一致 click 和 event.
  • sender: 事件源
  • args: 事件参数

下面的例子创建了一个 InputNumber 控件，并添加了 valueChanged 事件处理，使用对话框呈现控件的新值。

<!-- HTML -->
<div data-bind="wjInputNumber: { value: theValue, step: 1, valueChanged: valueChangedEH }"></div>
 
//View Model
this.valueChangedEH = function (data, sender, args) {
    alert('The new value is: ' + sender.value);
}

同样的，上述例子中的控件可以使用定制元素语法定义如下:

<wj-input-number value="theValue" step="1" value-changed="valueChangedEH"></wj-input-number>

绑定未定义监控属性

视图模型的监控属性指定到 undefined 值时，Wijmo绑定在初始化处理时会进行特殊处理。例如，你像如下这种方式绑定控件属性，ko.observable(undefined) 或 ko.observable()，Wijmo不会为控件指定值，相反的，在属性支持双向绑定时，将使用控件默认值初始化绑定监控属性， 因为在绑定完成初始化后，绑定会将这类型属性的控件值更新到绑定的视图模型的监控属性上。注意 null 值，例如 ko.observable(null), 会使用一般的方式处理，即为绑定的控件属性设置 null 值。 在这一重要的初始化过程结束后，对于 undefined 的监控属性, 反过来 Wijmo 会使用一般的方式处理,并使用 undefined 设置相应的控件属性。

在下边的例子中, InputNumber 控件的 value 属性拥有默认值0，在初始化后，相同的值会被指定到视图模型的 theValue : 

<!-- HTML -->
<div data-bind="wjInputNumber: { value: theValue }"></div>
 
//View Model
this.theValue = ko.observable();

定义对象属性和数组属性

一些 Wijmo 控件拥有数组属性或对象属性。 例如 FlexChart 控件公开了 axisX 和 axisY 属性，其定义为 Axis 对象; 还有一个包含了 Series 对象的数组属性 series 。 Wijmo 位这类控件提供了特殊的绑定方式，以便我们能够为这些控件元素的子元素设置绑定。如果控件暴漏了多个拥有相同类型定义的对象属性，需要使用 wjProperty 属性来明确指定绑定的对象属性。

下边的例子定义了一个 FlexChart 控件，拥有 axisX 和 axisY 对象属性，以及两个 series 对象属性:

<div data-bind="wjFlexChart: { itemsSource: data, bindingX: 'country' }">
    <div data-bind="wjFlexChartAxis: { wjProperty: 'axisX', title: chartProps.titleX }"></div>
    <div data-bind="wjFlexChartAxis: { wjProperty: 'axisY', title: chartProps.titleY }"></div>
    <div data-bind="wjFlexChartSeries: { name: 'Sales', binding: 'sales' }"></div>
    <div data-bind="wjFlexChartSeries: { name: 'Downloads', binding: 'downloads' }"></div>
</div>

上述例子中的控件可以使用定制元素语法定义如下::

<wj-flex-chart items-source="data" binding-x="'country'">
    <wj-flex-chart-axis wj-property="'axisX'" title="chartProps.titleX"></wj-flex-chart-axis>
    <wj-flex-chart-axis wj-property="'axisY'" title="chartProps.titleY"></wj-flex-chart-axis>
    <wj-flex-chart-series name="'Sales'" binding"'sales'"></wj-flex-chart-series>
    <wj-flex-chart-series name="'Downloads'" binding"'downloads'"></wj-flex-chart-series>
</wj-flex-chart>

control 属性

每一个 Wijmo 控件绑定都公开了一个 control 属性，指向绑定创建的 Wijmo 控件实例。这样就可以在视图模型的代码或其他绑定中方便的引用到该控件。 例如，下边的标签创建了一个 FlexGrid 控件，它的引用存储在视图模型的 flex 监控属性中，在按钮的点击事件处理中使用该引用将表格移动到下一个记录行:

<!-- HTML -->
<div data-bind="'wjFlexGrid': { itemsSource: data, control: flex }"></div>
<button data-bind="click: moveToNext">Next</button>
 
//View Model
this.flex = ko.observable();
this.moveToNext = function () {
    this.flex().collectionView.moveCurrentToNext();
}

initialized 事件

每一个 Wijmo 控件绑定都公开了一个 initialized 事件以及一个 isInitialized 属性。该事件在绑定完成创建控件并使用标签属性指定的值完成初始化后被触发。 因为绑定可以包含自绑定，例如 wjFlexGrid 包含子绑定 wjFlexGridColumn ，这也意味着子绑定也完成了初始化并反映给了父绑定。isInitialized 属性在触发 initialized 事件之前会被设置为true, 你可以在视图模型中定义一个绑定 isInitialized 属性的监控属性用来观察它的值的变化。

下边的例子在控件完成初始化后调整 FlexGridColumn 的格式，可以肯定的是这些格式设置不会被绑定定义的格式覆盖：

<!-- HTML -->
<div data-bind="'wjFlexGrid': { itemsSource: dataArray, initialized: flexInitialized }">
     <div data-bind="wjFlexGridColumn: { binding: 'sales', format: 'n2' }"></div>
     <div data-bind="wjFlexGridColumn: { binding: 'downloads', format: 'n2' }"></div>
</div>
 
//View Model
this.flexInitialized = function (data, sender, args) {
    var columns = sender.columns;
    for (var i = 0; i < columns.length; i++) {
        if (columns[i].dataType = wijmo.DataType.Number) {
            columns[i].format = 'n0’;
        }
    }
}

定义元素预处理器

Wijmo Knockout 预处理器使用标准的Knockoutko.bindingProvider.instance.preprocessNode API. 同一页面中如果有其他的定制预处理器时，这可能会导致一些问题， 因为 Knockout 仅提供了单一的实例属性用来指向预处理器函数，这样后注册的预处理器会覆盖掉之前的 。

为了确保附加的其他预处理有效，Wijmo Knockout 预处理初始化时，会存储当前已注册的预处理器，并在需要处理不能识别为Wijmo控件的的元素时委托调用，这样的组织形成可以理解为一个预处理器栈。但是如果你在 Wijmo Knockout 预处理器之后注册了其他的预处理器 (即其他预处理器 <script> 脚本应用在 wijmo.knockout.js 模块之后执行)，那么你需要确保其它的预处理器有类似预处理器栈的机制，否则 Wijmo konckout 预处理器会失效。如果你确实需要让 Wijimo konckout 预处理器失效，那么设置 wijmo.disableKnockoutTags 属性为 false 即可，注意需要在引用core Wijmo 模块之后，wijmo.knockout.js 模块之前进行设置，例如:

<script src="scripts/wijmo.js" type="text/javascript"></script>
<script src="scripts/wijmo.input.js" type="text/javascript"></script>
 
<script type="text/javascript">
    wijmo.disableKnockoutTags = true;
</script>
<script src="scripts/wijmo.knockout.js" type="text/javascript"></script>

注意在这种场景下只能使用常规的绑定语法，Wijmo的定制标记不会被识别。

        header: 'Revenue', 
        binding: 'amount', 
        format: 'n0' }">
    </div>
    <div data-bind="wjFlexGridColumn: { 
        header: 'Active', 
        binding: 'active' }">
    </div>
</div>

wjFlexGridColumn 支持 Column 类的所有可读写属性及事件。isSelected 属性支持双向绑定。

可以向 Column 类的属性添加匹配的正则式，一个 wjFlexGridColumn 绑定可能包含一个 wjStyle 绑定，该绑定可以为单元格设置条件格式化和 HTML 模板片段。表格行会自动验证填充定制单元格的内容。

wjStyle 绑定和 HTML 片段都可以使用 $item 监控属性变量，$item 监控属性在 KnockoutJS 绑定中是一个指向当前行的引用。 $row 和 $col 监控属性属性也是可用的，他们分别执行行列的索引，示例如下：

<div data-bind="wjFlexGridColumn: { 
        header: 'Symbol', 
        binding: 'symbol', 
        readOnly: true, 
        width: '*' }">
  <a data-bind="attr: { 
        href: 'https://finance.yahoo.com/q?s=' + $item().symbol() }, 
        text: $item().symbol">
  </a>
</div>
<div data-bind="wjFlexGridColumn: { 
     header: 'Change',
        binding: 'changePercent',
        format: 'p2',
        width: '*'
        },
        wjStyle: { 
        color: getAmountColor($item().change) }">
</div>

上述的绑定创建了两列。 第一列拥有一个模板，可以基于绑定的"symbol"属性生成一个链接； 第二列拥有一个条件样式，使用controller中定义的函数决定渲染value值时使用的颜色。
