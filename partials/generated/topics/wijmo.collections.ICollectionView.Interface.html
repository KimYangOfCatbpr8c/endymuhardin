使集合具有当前记录管理，自定义排序，过滤和归类的功能。

这是一个被用于在Microsoft XAML平台中的ICollectionView 接口的JavaScript版本。它提供了一个连续的，有力的并且MVVM友好地方式来将数据绑定到用户界面元素上。

Wijmo包括各种实现ICollectionView 的类。最常见的是CollectionView，它基于有规律的JavsScript数组进行工作。

属性

名称 类型 描述
F canFilter boolean 获取一个值，该值指示此视图是否支持通过filter 属性进行过滤。
F canGroup boolean 获取一个值，该值指示此视图是否支持通过groupDescriptions 属性进行分组归类。
F canSort boolean 获取一个值，该值指示此视图是否支持通过sortDescriptions 属性进行排序。
F currentChanged Event 在当前项目更改后发生。
F currentChanging Event 在当前项目更改前发生。
F currentItem any 获取视图中当前的条目。
F currentPosition number 获取视图中当前条目的顺序位置。
F filter IPredicate

获取或设置用于确定一个项目是否适合包含在视图中的回调函数

注意：如果过滤函数需要一个范围（例如，一个有意义的'this'值），然后请记住使用“绑定”函数指定“this”对象来设置过滤器。例如： 

  collectionView.filter = this._filter.bind(this);

F groupDescriptions ObservableArray 获取GroupDescription 对象的集合，该集合描述了如何在视图中将集合中的条目进行分组。
F groups any[] 获取高水平的分组。
F isEmpty boolean 获取一个值，该值指示该视图是否包含任何条目。
F items any[] 获取视图中的被过滤的，被排序的，被分类的项目。
F sortDescriptions ObservableArray 获取SortDescription 对象的集合，该集合描述了如何在视图中将集合的条目进行排序。
F sourceCollection any 获取或者设置集合对象从中创建视图。

方法

名称 参数 描述
beginUpdate​() 直到下一次调用endUpdate之前暂停刷新。
contains​(item): boolean

item (any): 条目在集合中进行定位。

 返回一个值，该值指示给定的条目是否属于这个视图。
deferUpdate​(fn)

fn (Function): 在一个beginUpdate/endUpdate块中被执行的函数。

在一个beginUpdate/endUpdate块中执行一个函数。

在执行功能已被执行之前，该集合将不会被刷新。 该方法保证endupdate被调用甚至如果函数抛出所谓。
endUpdate​() 通过调用beginUpdate 来重新开始暂停的刷新。
moveCurrentTo​(item): boolean

item (any): 将条目设置为currentItem.

 将指定条目设置为视图中的当前项。

moveCurrentToNext​(): boolean 将当前项后的项目设置为当前项目。

index (number): The index of the item to set as the currentItem.

 将该项在视图中的指定索引处设置为当前项目。
moveCurrentToPrevious​() 将视图中当前项之前的项目设置为当前项目。
refresh​() 重新创建使用当前排序、筛选器和分组参数的视图。
