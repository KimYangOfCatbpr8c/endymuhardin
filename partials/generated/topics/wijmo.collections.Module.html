模块 wijmo.collections

文件
wijmo.js

定义与数据相关的接口和类，包括ICollectionView 接口和CollectionView 类，还有ObservableArray 类。

类

名称 描述
ArrayBase 通知数组类的基类。
CollectionView

实现该ICollectionView 接口的类，在正规的JavaScript数组中暴漏数据。

CollectionView 类实现下面的接口：

  • ICollectionView: 提供当前的记录管理，自定义的排序，过滤，和分组归类。
  • IEditableCollectionView: 提供编辑，添加和移除项目的方法。
  • IPagedCollectionView: 提供分页功能。

为了使用CollectionView 类， 通过将其声明为一个数据源，并通过一个规则数组开始。然后使用 filter， sortDescriptions， groupDescriptions 和pageSize 属性配置视图。最后，使用items 属性访问视图。例如： 

  // 创建一个新的CollectionView 
  var cv = new wijmo.collections.CollectionView(myArray);
  // 按降序的数量排序项目
  var sd = new wijmo.collections.SortDescription('amount', false);
  cv.sortDescriptions.push(sd);
  // 显示数量只大于100的项目
  cv.filter = function(item) { return item.amount > 100 };
  // 在控制台里显示排序的过滤后的结果
  for (var i = 0; i < cv.items.length; i++) {
    var item = cv.items[i]; 
    console.log(i + ': ' + item.name + ' ' + item.amount);
  }

CollectionViewGroup 表示CollectionView对象为基础的groupDescriptions属性创建的组。
GroupDescription

表示类型定义分组条件的基类。

普遍用作这个PropertyGroupDescription目的的具体类。
NotifyCollectionChangedEventArgs 为collectionChanged 事件提供数据。
ObservableArray

发送通知更改的数组。

类引发 collectionChanged 事件，当更改与推送、弹出、拼接、插入或删除方法一起使用时。

警告：直接将值分配给数组成员或者数组长度的变化不引发collectionChanged 事件。
PageChangingEventArgs 为pageChanging 事件提供数据。
PropertyGroupDescription

以属性名为标准来描述项目分组。

例如，下面的代码会引起致CollectionView 通过它们的‘country’属性的第一个字母来进行分组: 

var cv = new wijmo.collections.CollectionView(items);
var gd = new wijmo.collections.PropertyGroupDescription('country');
cv.groupDescriptions.push(gd);

您还可以指定一个回调函数，该函数生成该组名。例如，下面的代码会引起致CollectionView 通过它们的‘country’属性的第一个字母来进行分组： 

var cv = new wijmo.collections.CollectionView(items);
var gd = new wijmo.collections.PropertyGroupDescription('country', 
  function(item, propName) {
    return item[propName][0]; // return country's initial
});
cv.groupDescriptions.push(gd);

SortDescription 描述一个排序标准。

接口

名称 描述
ICollectionView

使集合具有当前记录管理，自定义排序，过滤和归类的功能。

这是一个被用于在Microsoft XAML平台中的ICollectionView 接口的JavaScript版本。它提供了一个连续的，有力的并且MVVM友好地方式来将数据绑定到用户界面元素上。

Wijmo包括各种实现ICollectionView 的类。最常见的是CollectionView，它基于有规律的JavsScript数组进行工作。
IComparer 表示比较对象的方法。
IEditableCollectionView 定义扩展的ICollectionView以提供具有编辑功能的方法和属性。
INotifyCollectionChanged 通知动态变化的监听，例如, 条目增加并且移除或者当集合被排序，过滤或者归类时。
IPagedCollectionView 定义扩展的ICollectionView以提供具有翻页功能的方法和属性。
IPredicate 表示一种方法，该方法需要一个任何类型的项，并返回一个布尔值，指示对象是否符合一组标准。
