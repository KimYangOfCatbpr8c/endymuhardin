Clipboard

提供剪贴板操作的实用方法的静态类。

Clipboard 类提供了静态 copy 和paste 方法，它可以使用控件来自定义剪贴板操作中的剪贴板内容。

例如，下面的代码演示了如何控制可以截取剪贴板快捷键，并提供自定义剪贴板处理：

rootElement.addEventListener('keydown', function(e) {
  // 拷贝： ctrl+c 或者 ctrl+Insert
  if (e.ctrlKey && (e.keyCode == 67 || e.keyCode == 45)) {
    var text = this.getClipString();
    Clipboard.copy(text);
    return;
  }
  // 粘贴： ctrl+v 或者 shift+Insert
  if ((e.ctrlKey && e.keyCode == 86) || (e.shiftKey && e.keyCode == 45)) {
    Clipboard.paste(function (text) {
      this.setClipString(text);
    });
    return;
  }
});
Color

颜色类。

Color 类指定CSS字符串来解析颜色，并且暴露了它们的红，绿，蓝，和alpha通道作为读写属性。

Color 类也提供了fromHsb 和fromHsl 方法，使用HSB和HSL颜色模型代替RGB创建颜色。 并且用see:getHsb 和getHsl 方法来检索使用这些颜色模型的颜色组件。

最后，Color 类提供了一个interpolate 方法来通过插值使用HSL模型和两种颜色之间 创建颜色。这种方法是特别有用的，用于创建彩色动画的方法。
Control

所有Wijmo控件的基础类。

Control 类处理DOM元素和实际控件之间的联系。使用hostElement 属性获取托管在控件上的DOM元素，或者 getControl 方法获取托管在给定DOM元素上的控件。

Control 类也为无效和刷新控件，改变尺寸更新控件布局，处理定义在控件结构上的HTML模板提供了公共的模式。

Event

描述一个事件。

Wijmo 事件相似于.NET 事件。任何类可能将它们声明为字段来定义事件。任何类可以使用事件的addHandler 方法来订阅事件，也可以使用removeHandler 方法取消订阅。

Wijmo 事件处理器有两个参数: sender 和 args. 第一个参数是引发事件的对象，而第二个参数是包含事件参数的对象。

根据.NET模型为每一个事件定义事件的类。on[EVENTNAME] 方法引发事件。该模式允许衍生的来重载on[EVENTNAME] 方法，并且在引发基础类之前或者之后处理事件。 衍生的类可以甚至通过不调用基础类来阻止事件的实现。

例如，下面类脚本化的代码重载控件的 onValueChanged 事件，在执行一些处理之前且在valueChanged事件触发之后：

// 重载基础类
onValueChanged(e: EventArgs) {
  // 在事件触发前执行一些代码
  console.log('about to fire valueChanged');
  // 可选择地，调用基础类来触发事件
  super.onValueChanged(e);
  // 在事件触发之后执行一些代码
  console.log('valueChanged event just fired');
}

PrintDocument

类可以创建自定义文档的打印。

PrintDocument 类可以创建打印或输出为PDF文件。大多数浏览器允许你选择纸张大小，方向，空间，以及是否包含页眉和页脚。

为了使用，实例化一个PrintDocument，使用append 方法添加内容，并且通过调用print 方法来完成。

例如： 

// 创建文档
var doc = new wijmo.PrintDocument({
  title: 'PrintDocument Test'
});



// 添加一些简单的文本
doc.append('
Printing Example
');
doc.append('
This document was created using the PrintDocument class.
');



// 添加一些存在的元素
doc.append(document.getElementById('gaugeControl'));



// 打印文档(或者输出它为 PDF)
doc.print();

Tooltip

提供一个弹出窗口，该窗口显示页面上的其他元素的附加信息。

Tooltip 类可以在两个模式中使用：

Automatic 模式： 使用setTooltip 方法连接Tooltip 到页面上的一个或者更多个元素。 Tooltip 自动监听事件，并且当用户执行操作时触发提示信息。例如：

var tt = new wijmo.Tooltip();
tt.setTooltip('#menu', 'Select commands.');
tt.setTooltip('#tree', 'Explore the hierarchy.');
tt.setTooltip('#chart', '#idChartTooltip');

Manual 模式： 调用者使用show 和hide 方法负责显示和隐藏工具提示。例如：

var tt = new wijmo.Tooltip();
element.addEventListener('click', function () {
  if (tt.isVisible) {
    tt.hide();
  } else {
    tt.show(element, 'This is an important element!');
  }
});
TooltipEventArgs 为popup 事件提供参数。

var element = document.getElementById('someElement');

var element = document.getElementById('someElement');
if (this._animInterval) {
  clearInterval(this._animInterval);
}
var self = this;
self._animInterval = animate(function(pct) {
  element.style.opacity = pct;
  if (pct == 1) {
    self._animInterval = null;
  }
}, 1000);

var data = { name: 'Joe', amount: 123456 };
var msg = wijmo.format('Hello {name}, you won {amount:n2}!', data,
            function (data, name, fmt, val) {
              if (wijmo.isString(data[name])) {
                  val = wijmo.escapeHtml(data[name]);
              }
              return val;
            });

toFixed​(value, prec, truncate): number

value (number): 值进行四舍五入或者截取。
prec (number): 结果的若干小数位数。
truncate (boolean): 是否截取或者四舍五入原始值。

 将一个数字进行四舍五入或者截取到一个指定的精度。
toggleClass​(e, className, addOrRemove)

e (HTMLElement): 将有一个类被添加到元素上。
className (string): 类添加到元素上或者从该元素移除。
addOrRemove (boolean): 是否添加或者移除一个类。 使用true表示为元素添加类，而使用false表示从该元素移除一个类。

 给一个元素添加一个类，或者从该元素中移除一个类。
toHeaderCase​(text): string

text (string): 字符串转换为头类型。

通过首字母大写，将驼峰格式的字符串转换为头类型的字符串，并且在大写字母之前小写字符之后添加空格。

例如，‘somePropertyName’变为‘Some Property Name’。
tryCast​(value, type): any

value (any): 值进行类型转换。
type (any): 类型或者接口名称进行类型转换。

 如果可能的话，将一个值转换为一个类型。
返回值 如果类型转换成功，该值被传递，否则为null。
